# wifi_core 学生指南：板子怎么连上 Wi-Fi

适用板子：HiSpark WiFi-IoT（Hi3861 / OpenHarmony 迷你系统）  
阅读目标：会改账号密码、会编译烧录、看得懂串口日志、明白「连上网」分几步。

---

## 0. 先看结果：怎样算成功？

串口出现类似下面这几行，就说明 **连网成功**：

```text
WiFi: Connected, starting DHCP
STA IP: 192.168.250.95
STA Gateway: 192.168.250.203
STA Netmask: 255.255.255.0
[wifi_core_demo] OK, STA IP=192.168.250.95
[wifi_core_demo] still online
```

记住三个数的含义：

| 打印项 | 正常长什么样 | 含义 |
|--------|--------------|------|
| STA IP | `192.168.x.x` 一类 | 板子在局域网里的地址（最重要） |
| Gateway | 常为路由器 IP | 上网/出网关时走谁 |
| Netmask | 多为 `255.255.255.0` | 哪些地址算「同一局域网」 |

下面这些 **不算成功**（以前踩过的坑）：

```text
STA IP: 255.255.255.0     ← 这是掩码形态，不是板子 IP
STA Gateway: 254.128.0.0 ← 乱值
```

原则：**CONNECTED 只说明「连上热点」；真正能上网业务，必须等到拿到正常的 STA IP。**

## 1. 这个实验在干什么？（一句话）

让板子当手机一样去连教室路由器（**STA 模式**），拿到 IP，之后才能做 MQTT、Socket 等联网实验。

`wifi_core` = 把「连 Wi-Fi」做成可复用小模块。  
业务代码只调用几个函数，不要每个实验都复制一整段 `hi_wifi_*`。

---

## 2. 你会碰到的名词

| 名词 | 白话 |
|------|------|
| STA | Station，板子当「终端」去连别人的热点（最常见） |
| SoftAP | 板子自己当热点，别人连它（本实验先不管） |
| SSID | 热点名字，如 `xmob` |
| 密码 / PSK | WPA2 密码 |
| DHCP | 路由器自动给板子分配 IP（不用手写 IP） |
| netif | 网卡口在软件里的对象，上面挂着 IP/网关/掩码 |
| 事件回调 | Wi-Fi 驱动异步通知你：「扫完了」「连上了」「掉线了」 |

分层直觉（从上到下）：

```text
你的实验代码（wifi_core_demo / day08_mqtt）
        ↓ 只调用 WifiApp_xxx
wifi_core（wifi_connect.c）
        ↓ 内部用芯片 Wi-Fi API
芯片 SDK + lwIP（扫描/连接/DHCP/拿 IP）
```

## 5. 连网原理：像过安检的四步

```text
① 启动 STA          hi_wifi_sta_start
②（可选）扫描热点     看串口打印有没有你的 SSID
③ 发起连接           带上 SSID + 密码
④ 等 DHCP 拿 IP     CONNECTED 之后还要再等一会儿
        ↓
   拿到 192.168.x.x  → 才算 WifiApp_IsReady() == 真
```

对应 `wifi_core_demo` 里就是三句话：

```c
WifiApp_StartSta();                 // 开始：扫描 + 连接
WifiApp_WaitReady(60000);           // 最多等 60 秒，直到有合法 IP
WifiApp_GetIpString(ip, sizeof(ip)); // 把 IP 读出来打印
```

异步时序（为什么要 `WaitReady`，而不是 `StartSta` 一返回就上 MQTT）：

```text
时间 →

StartSta() 返回
   │  此时往往还没 IP！
   │
   ├── 驱动：CONNECTED（连上热点）
   ├── 启动 DHCP
   ├── ……几秒……
   └── 拿到 STA IP=192.168.x.x  ← WaitReady 在这里才返回成功
```

口诀：**「连上热点」≠「拿到 IP」≠「MQTT 一定通」**。  
业务必须等 `WaitReady` 成功后再连云端。

## 6. 四个 API（业务只认这些）

头文件：`wifi_connect.h`

| 函数 | 作用 |
|------|------|
| `WifiApp_StartSta()` | 启动并按配置去连热点 |
| `WifiApp_WaitReady(ms)` | 等到有合法 IP；超时返回失败 |
| `WifiApp_IsReady()` | 现在还有没有 IP（掉线会变假） |
| `WifiApp_GetIpString(buf,len)` | 拷贝 IP 字符串，方便打印 |

业务模块（MQTT、传感器上报）原则：

- `#include "wifi_connect.h"`  
- **不要** `#include "hi_wifi_api.h"`（除非进阶专题）


```gn

```

然后按课程流程编译、烧录，打开串口（常见 115200）。

## 8. 串口日志怎么读？

按时间顺序对照：

| 日志 | 含义 |
|------|------|
| `[wifi_core_demo] WifiApp_StartSta...` | 应用开始连网 |
| `SSID: xmob` … | 扫描结果；里面应有你的热点名 |
| `WiFi: connecting SSID="xmob"` | 正在连这个名字 |
| `+NOTICE:CONNECTED` / `WiFi: Connected, starting DHCP` | 已关联热点，开始要 IP |
| `STA IP: 192.168.x.x` | DHCP 成功（验收关键） |
| `[wifi_core_demo] OK, STA IP=...` | demo 侧确认成功 |
| `still online` | 周期性检查：还在线 |
| `wait IP timeout` / `StartSta failed` | 失败，查下表 |

常见排查：

| 现象 | 可能原因 |
|------|----------|
| 扫描列表没有你的 SSID | 热点关了 / 2.4G 才支持 / 距离远 |
| 有 SSID 但不 CONNECTED | 密码错、加密方式错 |
| CONNECTED 但一直等不到 IP / 超时 | 路由器 DHCP 异常；或仍用旧固件有取 IP bug |
| BUILD.gn 报 Expected comma | feature 字符串少了引号 |

## 9. `wifi_core_demo.c` 在干什么？（对照源码读）

```c
WifiCoreDemo_Entry()          // SYS_RUN：开机自动进这里
  → 创建线程 WifiCoreDemo_Task
       → StartSta / WaitReady / 打印 IP
       → 死循环里每秒 IsReady，打印 still online / offline
```

它 **没有** MQTT、没有传感器，专门验证：`wifi_core` 能不能稳定拿到 IP。  
下一步做 Day08 MQTT 时：连网仍用 `wifi_core`，只是 `WaitReady` 成功后改去发 MQTT。

## 12. 还想看源码时，优先看这些函数

在 `wifi_connect.c` 里：

| 函数 | 读什么 |
|------|--------|
| `WifiApp_StartSta` | 启动 STA、找网卡、扫描、发起连接 |
| `WifiApp_EventCb` | CONNECTED 时启动 DHCP；掉线清理 |
| `WifiApp_IsReady` / `WaitReady` | 轮询，直到 DHCP 给出合法 IPv4 |
| `WifiApp_FetchIpv4` | 用 `netifapi_netif_get_addr` 正确取 IP（不要直接乱读结构体字段） |
