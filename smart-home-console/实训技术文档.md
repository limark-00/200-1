## 1. 这个项目解决什么问题

实训里通常有三块东西，平时是分开的：

| 东西                      | 平时怎么玩          | 痛点           |
| ----------------------- | -------------- | ------------ |
| Hi3861 板子（灯、蜂鸣器、光敏、PIR） | 经 **巴法云** 收发消息 | 要开控制台手动看、手动发 |
| 电脑摄像头                   | 另写 OpenCV 脚本   | 和物联网对不上      |
| 人                       | 想看一个网页就搞定      | 没有统一入口       |

本项目做一个 **网页控制台**，把它们串起来：

```text
你（浏览器）
    ↕  HTTP（网页 + /api/...）
FastAPI 后端（python app.py）
    ├─→ bemfa_api.py  →  巴法云  →  Hi3861 板子
    └─→ camera.py     →  电脑摄像头 → static/captures/ 照片
```

**一句话：用网页查看传感器、控制灯/蜂鸣器，并在「有人」时自动拍照。**

## 2. 先建立一张「总图」

### 2.1 一次「点红灯」走什么路

```text
点击「红灯」
  → static/script.js 发 POST /api/control  {"topic":"trafficLight","msg":"red"}
  → app.py 收到，调用 bemfa_api.send_msg(...)
  → 模拟模式：只改内存，假装成功
  → 真实模式：请求巴法云 sendMessage
  → 板子订阅了该主题 → 收到 "red" → 亮红灯
```

### 2.2 一次「有人就拍照」走什么路

```text
板子 PIR 检测到人 → 向 PIR 主题发 "1"
  → app.py 后台线程每 3 秒查询 PIR 主题
  → 判定为「触发」→ camera.capture_photo()
  → 图片存到 static/captures/时间戳.jpg
  → 网页照片墙定时 GET /api/captures 刷新显示
```

没有板子时：点「模拟 PIR 触发」= 假装板子发了一次「有人」。

### 2.3 文件结构

本项目按「职责」拆开，**你想改什么，就只动对应文件**：

| 你想改的东西           | 去哪个文件                  |
| ---------------- | ---------------------- |
| uid、主题名、端口、轮询秒数  | `config.py`            |
| 怎么访问巴法云、Mock 假数据 | `bemfa_api.py`         |
| 怎么拍照、识别占位        | `camera.py`            |
| 网页地址、API、PIR 轮询  | `app.py`               |
| 按钮、卡片布局          | `templates/index.html` |
| 颜色、排版            | `static/style.css`     |
| 定时刷新、点按钮发请求      | `static/script.js`     |

### 3.1 独立虚拟环境？

**发给学生：每人用 venv + `requirements.txt`。**


## 5. FastAPI 原理（结合 app.py）

FastAPI 可以想成「前台接待」：

1. 创建应用 `app = FastAPI(...)`  
2. 用 `@app.get` / `@app.post` 告诉它：哪个网址对应哪个函数  
3. 函数里做事，返回网页或字典（字典会自动变成 JSON）  
4. 用 uvicorn 启动，开始监听端口  

### 5.1 创建应用 + 生命周期

程序启动时要开 PIR 轮询线程，关闭时要停掉：

```python
@asynccontextmanager
async def lifespan(_app: FastAPI):
    _start_poller_once()   # 启动时
    yield
    _stop_poller()         # 关闭时

app = FastAPI(title="智能家居实训控制台", lifespan=lifespan)
```

### 5.2 返回网页

```python
@app.get("/")
async def index(request: Request):
    return templates.TemplateResponse(
        "index.html",
        {
            "request": request,   # FastAPI 模板必须传 request
            "topics": {...},
            "mock_mode": _get_mock_mode(),
        },
    )
```

静态文件（CSS/JS/图片）：

```python
app.mount("/static", StaticFiles(directory=STATIC_DIR), name="static")
```

所以浏览器访问 `/static/script.js`、`/static/captures/xxx.jpg`。

### 5.3 返回 JSON（API）

```python
@app.get("/api/mode")
async def get_mode():
    return {"ok": True, "mock": _get_mock_mode()}
```

返回 Python 字典即可，FastAPI 自动转成 JSON。

### 5.4 用 Pydantic 读 POST 的 JSON

```python
class ControlBody(BaseModel):
    topic: str
    msg: str

@app.post("/api/control")
async def api_control(body: ControlBody):
    return bemfa_api.send_msg(body.topic.strip(), body.msg.strip())
```

好处：缺字段会自动报错；`/docs` 里能看到字段，方便试调。

### 5.5 路径参数

```python
@app.get("/api/sensor/{topic:path}")
async def api_sensor(topic: str):
    return bemfa_api.get_topic_msg(topic)
```

访问 `/api/sensor/tempHum` → 函数参数 `topic == "tempHum"`。

### 5.6 本项目 API 一览

| 方法   | 路径                    | 作用                          |
| ---- | --------------------- | --------------------------- |
| GET  | `/`                   | 控制台网页                       |
| GET  | `/api/mode`           | 查询是否模拟模式                    |
| POST | `/api/mode`           | 切换模式 `{"mock": true/false}` |
| GET  | `/api/sensor/{topic}` | 查某主题最新消息                    |
| POST | `/api/control`        | 下发控制 `{"topic","msg"}`      |
| GET  | `/api/captures`       | 最近照片文件名列表                   |
| POST | `/api/mock/pir`       | 模拟人体感应（仅模拟模式）               |
| POST | `/api/capture/now`    | 立刻拍一张                       |

### 5.7 后台为什么还要「轮询线程」？

网页接口是「有人访问才执行」。  
PIR 需要 **主动、定时** 问：「现在有人吗？」

所以 `app.py` 里有守护线程，大约每 `PIR_POLL_INTERVAL`（默认 3）秒查一次 PIR。  
还有 `CAPTURE_COOLDOWN`（默认 10 秒）：人一直站着也不能疯狂连拍。

## 6. 前端如何和后端交互

```text
浏览器里的 HTML/JS
    ↓  只允许访问本机 FastAPI（不要在 JS 里写巴法云私钥）
FastAPI
    ↓
巴法云 / 摄像头
```

### 6.1 页面从哪来？

`templates/index.html` 由后端渲染。  
后端把主题名注入给 JS：

```html
<script>
  window.TOPICS = {{ topics | tojson }};
  window.INITIAL_MOCK = {{ mock_mode | tojson }};
</script>
<script src="/static/script.js"></script>
```

`app.py` 里的 `tojson` 过滤器必须返回 `Markup`，否则引号会被转成 `&#34;`，  
JS 解析失败 → `TOPICS` 为空 → 请求变成 `/api/sensor/` → 报「topic 为空」。  
（这是 FastAPI + Jinja2 的常见坑，代码里已按正确方式写好。）

### 6.2 JS 怎么发请求？（核心是 fetch）

`static/script.js`：

```javascript
// 查询传感器（GET）
fetch("/api/sensor/" + encodeURIComponent(TOPICS.light))

// 下发控制（POST + JSON）
fetch("/api/control", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ topic: topic, msg: msg }),
})
```

每 5 秒自动刷新：

```javascript
setInterval(refreshSensors, 5000);
setInterval(refreshGallery, 5000);
```

### 6.3 照片墙怎么显示？

1. `GET /api/captures` 返回文件名列表  
2. 前端拼地址：`/static/captures/文件名`  
3. FastAPI 的 `StaticFiles` 负责把文件发出去  

### 6.4 右上角模式开关

开关拨到 **模拟** = `POST /api/mode` `{"mock": true}`  
拨到 **巴法云** = `{"mock": false}`  

「模拟 PIR 触发」**只在模拟模式可用**；巴法云模式下由设备真实上报。

## 7. 如何和巴法云交互

### 7.1 巴法云是什么角色？

把它想成 **邮局 / 公告栏**：

- **主题（topic）** = 频道名  
- 板子往主题发消息 = 张贴公告（例如光照 `320`、PIR `1`）  
- 我们的电脑读主题 = 看最新公告  
- 我们往主题发指令 = 贴「请开红灯」  
- 板子订阅了该主题就能收到并执行  

官方文档：https://cloud.bemfa.com/docs/

本项目用的 HTTP 接口（已封装）：

| 动作   | 地址                                      |
| ---- | --------------------------------------- |
| 获取消息 | `https://apis.bemfa.com/va/getmsg`      |
| 推送消息 | `https://apis.bemfa.com/va/sendMessage` |

### 7.2 配置写在 config.py

```python
BEMFA_UID = "你的私钥"
BEMFA_TYPE = 1    # 1=MQTT，3=TCP，必须和创建主题时一致

TEMP_HUM_TOPIC = "你的温湿度主题"
LIGHT_TOPIC = "你的光照主题"
PIR_TOPIC = "你的人体感应主题"
TRAFFIC_LIGHT_TOPIC = "你的交通灯主题"
BUZZER_TOPIC = "你的蜂鸣器主题"
RGB_TOPIC = "你的RGB主题"
```

### 7.3 代码封装在 bemfa_api.py

| 函数                         | 作用            |
| -------------------------- | ------------- |
| `get_topic_msg(topic)`     | 读最新一条消息       |
| `send_msg(topic, msg)`     | 下发一条指令        |
| `is_pir_triggered(msg)`    | 判断是否「有人」      |
| `set_mock_pir(True/False)` | 仅模拟模式：手动改 PIR |

统一返回格式（方便前端）：

```json
{"ok": true, "topic": "lightSensor", "msg": "320", "time": "...", "error": ""}
```

### 7.4 控制指令字符串谁说了算？

网页现在发的是：

- 交通灯：`red` / `yellow` / `green` / `off`  
- 蜂鸣器、RGB：`on` / `off`  

这些必须和 **板端固件 if 判断的字符串** 完全一致。  
云平台只负责「把字符串送到主题」，不理解「红灯」是什么意思。

PIR「有人」默认认：`1`、`on`、`触发`、`有人` 等。  
若你的板子发的是别的（如 `PIR=1`），只改 `bemfa_api.is_pir_triggered()`。

## 8. 如何调用摄像头

### 8.1 原理

```python
import cv2
cap = cv2.VideoCapture(0)   # 0=默认摄像头
ok, frame = cap.read()
cv2.imwrite("xxx.jpg", frame)
cap.release()
```

封装在 `camera.capture_photo()`，文件名用时间戳，目录 `static/captures/`。

### 8.2 谁会调用它？

1. **自动**：PIR 轮询发现有人 → `_try_capture_on_pir()`  
2. **手动**：页面「立即抓拍」→ `POST /api/capture/now`  

### 8.3 没有摄像头怎么办？

会生成一张灰色占位图，保证「流程」仍能测通（照片墙能看到文件）。

外接摄像头不是 0 号时，可改：

```python
camera.capture_photo(camera_index=1)
```

### 8.4 图像识别预留

```python
def recognize_image(image_path: str) -> dict:
    # TODO: 以后接大模型 / YOLO 等
    return {"description": "识别功能开发中（假返回值）"}
```

网页「识别功能开发中」对应这里。以后做 AI，优先改这个函数。

## 9. 模拟模式 vs 巴法云模式

|       | 模拟模式          | 巴法云模式          |
| ----- | ------------- | -------------- |
| 传感器数据 | 内存假数据（会轻微抖动）  | 真实 getmsg      |
| 控制下发  | 只改内存          | 真实 sendMessage |
| PIR   | 可用「模拟 PIR 触发」 | 等设备上报          |
| 适合    | 没板子、先练界面      | 联调硬件           |

`config.MOCK_MODE = True` 是**启动默认值**。  
页面开关可立即切换；**重启后又回到 config 的值**。



## 10. 常见修改任务

### 任务 A：改成自己的巴法云主题

1. 登录 https://cloud.bemfa.com/  
2. 查看/创建主题，记下名字和类型（MQTT=1 / TCP=3）  
3. 编辑 `config.py` 里的 `BEMFA_UID`、`BEMFA_TYPE`、各 `*_TOPIC`  
4. 重启 `python app.py`，开关拨到「巴法云」测试  

### 任务 B：改控制指令（例如板子认的是 `LED_RED` 不是 `red`）

改 `templates/index.html` 按钮：

```html
<button ... data-control="traffic" data-msg="LED_RED">红灯</button>
```

`data-msg` 会原样发给 `/api/control`。

### 任务 C：改 PIR「有人」的判定字符串

编辑 `bemfa_api.py` 的 `is_pir_triggered()`：

```python
trigger_words = {"1", "on", "触发", "有人", "你的板子实际字符串"}
```

### 任务 D：改轮询更快 / 少连拍

`config.py`：

```python
PIR_POLL_INTERVAL = 2   # 秒，查得更勤
CAPTURE_COOLDOWN = 15   # 秒，两次抓拍间隔更长
```

### 任务 E：改刷新速度（前端）

`static/script.js` 末尾：

```javascript
setInterval(refreshSensors, 3000);  // 改成 3 秒
```

### 任务 F：加一个新按钮（例如「全关」）

1. 在 `index.html` 加按钮，带 `data-control` / `data-msg`  
2. 若是新设备，在 `config.py` 加 topic，在 `script.js` 的 `sendControl` 映射表里加一项  
3. 板端固件也要认这个消息  

### 任务 G：接入真正的图像识别

只改 `camera.recognize_image(image_path)`：读图片 → 调 API → 返回结果。  
可在 `app.py` 的 `_try_capture_on_pir` 里把返回值存起来，再加一个 `/api/recognize/latest` 给前端显示。

### 任务 H：改页面颜色

改 `static/style.css` 顶部的 CSS 变量：

```css
:root {
  --accent: #0b6e99;   /* 主题色 */
  --bg: #e8eef4;       /* 背景 */
}
```

## 附录：最小知识地图

```text
config.py          配置（uid / topic / 端口 / Mock）
       ↓
app.py (FastAPI)   网页 + /api/* + /docs + PIR 轮询线程
       ↓
script.js          fetch 调 API，定时刷新界面
       ↓
bemfa_api.py       和巴法云说话（或 Mock 假数据）
camera.py          和摄像头说话
       ↓
板子执行 / 照片保存 / 网页展示
```

**FastAPI 负责开门营业并自动出说明书；API 是前后端约定；巴法云和板子传话；OpenCV 拍照。**

把这四句话搞懂，再带着问题去读源码注释，你会发现每个文件都在做一件很单纯的事——这也是你以后改功能时最该保持的习惯。
